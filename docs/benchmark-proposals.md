# 벤치마크 제안 — 레퍼런스 서비스에서 nuguzip이 가져올 수 있는 것

레퍼런스: 호갱노노 · 네이버부동산 · 직방 · 당근마켓(부동산) · 숨고 · 재개발닷컴
축: **A. 액션(전환·행동 유도)** / **B. 모션(움직임·전환 연출)** / **C. 연결성(화면·기능 간 동선)** / **D. 심화 콘텐츠**

전제(현실 제약):

- **매물 스크래핑 불가** — 네이버부동산·직방의 매물 데이터는 약관·저작권상 수집 불가. 매물 관련 제안은 모두 "실거래(국토부 공공데이터)·자체 콘텐츠(임장노트)·직접 제휴" 기반으로만 설계.
- 기존 자산: 실거래 41k건 · 시세 61지역 · 단지 42k개 · 임장노트 · AI 분석 · 모임/채팅. 제안은 전부 이 자산과 연결되는 것만 채택.
- 프로덕션 DB는 read-only 전제 — 신규 테이블이 필요한 항목은 난이도에 반영.

난이도: **즉시**(기존 데이터·컴포넌트로 며칠 내) / **단기**(1~2 스프린트) / **중기**(신규 데이터 파이프라인·설계 필요)

---

## A. 액션 — 행동 유도·전환

### A1. 지도 클러스터 가격 라벨 (호갱노노) — ✅ 이번에 구현

- **패턴**: 호갱노노 지도는 낮은 줌에서도 "가격"이 1급 정보. 클러스터 버블에 개수 대신 대표 시세를 노출해 "눌러보고 싶은 지도"를 만든다.
- **적용안**: `/api/map/clusters` 클러스터 응답에 셀 평균 실거래가(만원)를 추가하고, 마커를 "N개 · 12.3억" 알약 라벨로. 소스는 `market_complex_price` → 없으면 `complex_transactions`(실거래 캐시) 순 best-effort.
- **난이도**: 즉시 (구현 완료)
- **제약**: 시세 데이터가 없는 셀은 개수만 표시(폴백 필수). 호가가 아닌 실거래 기반임을 UI에 명시하는 것이 안전(허위 시세 표시 논란 방지).

### A2. 최근 본 단지 (직방·네이버부동산) — ✅ 이번에 구현

- **패턴**: 직방·네이버부동산 모두 "최근 본 방/단지"로 재방문 세션의 첫 행동을 단축.
- **적용안**: `/complex/[id]` 방문 시 localStorage(`nz_recent_complexes`, 최대 8) 기록 → `/search`에 "최근 본 단지" 칩 행. 로그인 불필요, 서버 부하 0.
- **난이도**: 즉시 (구현 완료)
- **제약**: 없음(로컬 저장). 추후 로그인 시 서버 동기화는 쓰기 필요 → 단기.

### A3. 관심 단지 가격 변동 알림 (호갱노노)

- **패턴**: "내 단지 실거래 떴어요" 푸시가 호갱노노 리텐션의 핵심.
- **적용안**: 이미 있는 user_watchlist(비교 트레이 서버 반영)와 실거래 41k 파이프라인을 연결 — 월간 실거래 갱신 시 워치리스트 단지에 신규 거래가 생기면 알림함(/notifications)에 적재. 이메일은 기존 digest 인프라 재사용.
- **난이도**: 단기 (알림 적재 테이블·cron은 기존 ETL 훅에 추가)
- **제약**: 국토부 실거래 공개 지연(계약 후 최대 30일)을 문구로 고지 필요. 푸시(FCM)는 중기.

### A4. "이 지역 전문가에게 물어보기" 견적형 CTA (숨고)

- **패턴**: 숨고는 콘텐츠 열람 → 견적 요청 폼 → 전문가 매칭의 3단 퍼널이 전부다.
- **적용안**: 단지 허브·AI 분석 하단에 "이 단지 관련 상담 요청" 폼(중개·세무·대출 카테고리 선택) → 기존 experts/모임·채팅으로 라우팅. 전문가 응답은 기존 채팅 사용.
- **난이도**: 단기 (전문가 풀이 얇으면 "질문 게시판" 폴백으로 시작)
- **제약**: 중개 알선 행위로 보이지 않도록 문구 주의(중개보수 관련 표시·광고 규제). 결제 연결 시 통신판매중개 고지 필요.

### A5. 임장노트 작성 부담 낮추기 — 현장 원탭 기록 (당근마켓)

- **패턴**: 당근은 판매글 작성을 사진 1장+제목으로 시작하게 해 공급 콘텐츠를 늘렸다.
- **적용안**: notes/new의 "빠른 기록 모드" — 단지 프리필(A7) + 체크 5항목만으로 30초 저장, 나머지는 나중에 보완. 이미 있는 임시저장(nz_note_draft)과 결합.
- **난이도**: 즉시~단기 (기존 폼의 축약 뷰)
- **제약**: 없음.

### A6. "지금 N명이 보는 중" 실시간 소셜 프루프 (직방·호갱노노)

- **패턴**: 매물 상세의 "N명이 보는 중"이 의사결정을 재촉한다. 현재 nuguzip은 목업 문자열로만 존재.
- **적용안**: 단지 허브 조회 이벤트를 경량 카운터(Redis/Supabase edge, 5분 윈도)로 집계해 실수치로 교체. 값이 0~1이면 숨김(역효과 방지).
- **난이도**: 단기
- **제약**: 쓰기 스토리지 필요(프로덕션 DB read-only 정책과 분리된 카운터 스토어 권장). 허수 표기는 표시광고법상 기만 소지 — 반드시 실측만.

## B. 모션 — 움직임·전환 연출

### B1. 지도 마커 등장/전환 애니메이션 (호갱노노)

- **패턴**: 호갱노노는 줌 전환 시 가격 라벨이 스르륵 나타나 지도가 "살아있게" 느껴진다.
- **적용안**: NaverMap 커스텀 HTML 마커에 globals.css의 riseIn/dur-200 그대로 인라인 적용(마커 content가 HTML이므로 CSS 애니메이션 클래스 삽입 가능). 클러스터↔포인트 모드 전환 시 페이드.
- **난이도**: 즉시 (새 라이브러리 불필요 — 기존 토큰만)
- **제약**: 마커 수백 개 동시 애니메이션은 저사양에서 프레임 저하 — 뷰포트 내 상위 N개만.

### B2. 페이지 전환 riseIn 일관화 — ✅ 이번에 일부 적용

- **패턴**: 토스·직방류 앱의 "모든 화면이 같은 리듬으로 뜬다"는 감각은 단일 entrance 모션의 일관 적용에서 온다.
- **적용안**: rise-in 누락 화면 점검·적용(이번에 legal/community·legal/fees 적용, 지도 목록 카드 hover lift 추가). 이후 신규 페이지 체크리스트에 "rise-in 적용" 항목화.
- **난이도**: 즉시 (구현 완료 + 지속 운영)
- **제약**: prefers-reduced-motion 대응은 globals.css 전역 처리 유지.

### B3. 숫자 카운트업 — 시세·지표 강조 (호갱노노·토스)

- **패턴**: 시세 숫자가 0→8.1억으로 차오르며 "데이터가 방금 계산됐다"는 인상을 준다.
- **적용안**: 단지 허브 지표 4카드·AI 분석 점수에 dur-600 카운트업(requestAnimationFrame 소형 훅, 라이브러리 금지 준수). 차트 draw-in도 dur-lg 토큰 재사용.
- **난이도**: 즉시~단기
- **제약**: 없음.

### B4. 시트/패널 제스처 전환 (직방·당근)

- **패턴**: 모바일 지도 하단 시트를 드래그로 절반↔전체 전환 — 지도와 목록을 오가는 비용이 0에 수렴.
- **적용안**: /map 상세 패널을 모바일에서 바텀시트화(3단 스냅: 핸들만/절반/전체). CSS transform+터치 이벤트로 구현, dur-300 이징 토큰 사용.
- **난이도**: 단기
- **제약**: 없음(구현 품질 리스크만 — 스크롤 충돌 테스트 필요).

### B5. 스켈레톤 크로스페이드 (네이버부동산)

- **패턴**: 지도 idle 후 데이터 로드까지 회색 라벨 스켈레톤 → 실데이터 크로스페이드.
- **적용안**: 클러스터 fetch(디바운스 350ms) 동안 이전 마커를 60% 오퍼시티로 유지 → 응답 시 dur-120 크로스페이드. 목록 패널도 동일.
- **난이도**: 즉시
- **제약**: 없음.

## C. 연결성 — 화면·기능 간 동선

### C1. 단지 → 임장노트 프리필 (당근 "이 물건 문의하기" 패턴) — ✅ 이번에 구현

- **패턴**: 당근은 상세에서 채팅 시작 시 물건 컨텍스트가 자동으로 실린다. 컨텍스트 수동 재입력이 없어야 전환된다.
- **적용안**: `/complex/[id]` CTA → `/notes/new?apt=단지명` 프리필 + "지도에서 보기" 역링크. (구현 완료 — notes/new가 apt 쿼리를 읽어 위치 카드·저장 payload에 반영)
- **난이도**: 즉시 (구현 완료). 후속: apt 대신 complexId를 넘겨 노트↔단지 FK 연결(단기).
- **제약**: 없음.

### C2. 임장노트 → 모임 — "이 단지 같이 임장 가요" (당근 모임)

- **패턴**: 당근은 게시글에서 동네 모임으로 자연 유입시킨다.
- **적용안**: 공개 임장노트 상세 하단에 같은 지역 모임(기존 town/groups) 카드 1~2개 + "이 단지 임장 모임 만들기" CTA(지역·단지명 프리필).
- **난이도**: 즉시~단기 (모임 데이터 이미 존재)
- **제약**: 없음.

### C3. 단지 허브를 모든 데이터의 허브로 — 역링크 그물 (네이버부동산)

- **패턴**: 네이버부동산 단지 페이지는 시세·학군·기사 등 모든 정보의 종착지이자 출발지다.
- **적용안**: 실거래 41k·시세 61지역·노트·모임·AI 분석 각 화면에서 단지명이 등장하는 모든 지점을 `/complex/[id]` 링크로(이미 resolveComplexHref 존재 — 적용 범위 확대). 반대로 단지 허브에 "이 단지가 속한 지역 리포트(/region)" 링크.
- **난이도**: 즉시~단기 (점진 적용)
- **제약**: 단지명 문자열 매칭 실패 시 링크 숨김(현 정책 유지 — mock으로 보내지 않기).

### C4. 비교 트레이 전역화 (호갱노노 단지 비교)

- **패턴**: 호갱노노 비교는 어디서든 담고, 담긴 개수가 항상 보인다.
- **적용안**: 기존 compare-tray(localStorage+user_watchlist)를 지도 목록·검색 결과·발견 피드 카드에도 "+비교" 버튼으로 노출, 담긴 개수 플로팅 배지 → /notes/compare. CompareTrayButton 재사용.
- **난이도**: 즉시
- **제약**: 없음.

### C5. 검색 → 지도 딥링크 (네이버부동산)

- **패턴**: 검색 결과에서 지도로 갈 때 해당 단지가 중심+선택 상태로 열린다.
- **적용안**: `/map?complex=[id]` 또는 `?lat=&lng=&z=` 쿼리를 map-client가 읽어 초기 center/selected 설정. 최근 본 단지 칩(A2)·단지 허브 "지도에서 보기"(C1)가 이 딥링크를 사용하면 연결 완성.
- **난이도**: 즉시~단기
- **제약**: 없음.

## D. 심화 콘텐츠

### D1. 단지별 실거래 히스토리 차트 + 층/면적 분해 (호갱노노)

- **패턴**: 호갱노노의 본체는 "국토부 실거래를 가장 보기 좋게" — 평형별·층별 분해와 장기 추이.
- **적용안**: 이미 있는 complex_transactions(월별 평균)를 84㎡ 환산·기간 선택(1/3/10년) 차트로. 실거래 41k 원본(market_transactions)에서 층·면적 분포 추가 집계.
- **난이도**: 단기 (데이터 이미 보유 — 프론트 중심)
- **제약**: 국토부 데이터 재가공은 출처 표기 시 허용. "시세"가 아닌 "실거래 통계"로 라벨링.

### D2. 재개발·재건축 단계 트래커 (재개발닷컴)

- **패턴**: 재개발닷컴은 구역별 사업 단계(추진위→조합→관리처분→이주)를 지도에 얹어 유료화까지 성공.
- **적용안**: 서울·경기 정비사업 공공데이터(각 지자체 고시·클린업시스템 공개자료)를 수집해 지도 레이어 + 단지 허브 배지("재건축 조합설립 단계"). 임장노트에 "정비구역 체크리스트" 템플릿 연결.
- **난이도**: 중기 (신규 수집 파이프라인, 지자체별 포맷 상이)
- **제약**: 고시문은 공공저작물이나 지자체별 이용조건 확인 필요. 사업 단계 오표기는 투자 피해로 이어질 수 있어 "고시일 기준·출처" 필수 표기.

### D3. 지역 리포트 자동 생성 — "이번 달 우리 동네" (호갱노노 이야기+분석)

- **패턴**: 호갱노노는 데이터 요약을 "읽을거리"로 포장해 체류를 만든다.
- **적용안**: 시세 61지역 × 월별 스냅샷으로 "관양동 6월: 거래 12건, 전월비 -1.2%, 최고가 갱신 단지 1곳" 자동 리포트를 기존 digest/AI 분석 인프라로 생성 → /region·이메일 다이제스트 재사용. 해당 지역 인기 임장노트 3개를 함께 배치(자체 콘텐츠 차별화).
- **난이도**: 단기 (데이터·AI 파이프라인 기존 보유)
- **제약**: AI 생성 문구에 투자 권유 표현 금지(자본시장법·표시광고 리스크) — "정보 제공" 톤 고정.

### D4. 임장노트 집단지성 요약 — 단지 평판 지표 (당근 동네생활 + 호갱노노 이야기)

- **패턴**: 호갱노노 "이야기"는 UGC를 단지 단위로 묶어 데이터가 못 주는 질감을 제공한다.
- **적용안**: 단지별 공개 임장노트의 체크 항목(채광·주차·소음 등)을 집계해 "이웃 실측 지표"(예: 주차 아쉬움 74%)로 단지 허브에 노출. 노트 3건 미만이면 미표시. 기존 AI 요약(aiBody)의 근거 데이터로도 사용.
- **난이도**: 단기 (노트 스키마에 이미 구조화 점수 존재)
- **제약**: 특정 단지 명예훼손성 집계 방지 — 표본 수 명시·소표본 숨김 필수.

### D5. 전문가 답변 아카이브 → SEO 콘텐츠 (숨고 고수 노하우)

- **패턴**: 숨고는 전문가 Q&A를 공개 아카이브로 쌓아 검색 유입을 만든다.
- **적용안**: A4의 상담 질문·답변 중 공개 동의분을 `/complex/[id]`·지역 페이지 하위에 Q&A 섹션으로 축적(이미 있는 "이야기" 탭 확장). 단지명+질문이 롱테일 검색어를 정확히 타격.
- **난이도**: 중기 (콘텐츠 볼륨이 관건 — 초기엔 에디토리얼 시딩)
- **제약**: 전문가 답변의 법률·세무 조언 책임 고지 필요. 개인정보(주소·호수) 마스킹.

### D6. 청약·분양 캘린더 연동 (직방 분양)

- **패턴**: 직방은 분양 정보로 "매물 없이도" 방문 이유를 만든다.
- **적용안**: 이미 있는 applyhome(청약홈 API) 데이터를 지도 레이어("분양 예정" 마커)와 지역 리포트(D3)에 통합. 관심 지역 청약 알림은 A3 알림 인프라 공유.
- **난이도**: 단기 (청약홈 공공 API 기연동 — 노출 확대만)
- **제약**: 청약홈 API 이용약관 내 재표시 허용 범위 준수, 모집공고 원문 링크 필수.
