# CODEF 부동산 데이터 연동

## 배경

사용자가 업로드한 12개 `.xlsx`(2026-07-17)는 **데이터가 아니라 CODEF(codef.io) API 명세**다.
각 파일은 한 개의 CODEF 상품(엔드포인트)의 입력·출력 필드·기관코드·인증방식을 정의한다.
따라서 "파일 내용을 DB에 넣는다"는 것은 이 명세대로 **CODEF를 호출해 실데이터를 받아 적재**하는 것을 의미한다.

이 명세들은 `lib/codef/endpoints.ts`에 코드로 옮겨져 있으며, 화면(`/data/records`)에서 연동 현황으로 노출된다.

## 12개 상품 요약

| 상품 | 기관코드 | 인증 | 대량수집 | 비고 |
|---|---|---|---|---|
| KB 시세정보 | 0011 | 보안문자 | ✅ | 면적·호별 매매/전세 상·하한 평균가, 협력 중개사 |
| 단지목록 조회 | 0011 | 공개 | ✅ | 지역별 단지 목록 |
| 단지 일련번호 | 0011 | 공개 | ✅ | 단지명→KB 일련번호(시세 키) |
| 단지 기본정보(K-apt) | kapt | 공개 | ✅ | 세대수·승인일 등, 주 1회 갱신 |
| 실거래가(아파트·연립·오피) | 0010 | 공개 | ✅ | 국토부 실거래(이미 MOLIT로 운영 중) |
| 실거래가(단독·다가구) | 0010 | 공개 | ✅ | 단독·다가구 실거래 |
| 공동주택 공시가격 | 0007 | 보안문자 | ✅ | 아파트 공시가격 |
| 개별주택 가격 | 0007 | 보안문자 | ✅ | 단독주택 공시가격 |
| 개별공시지가(일사편리) | 0001 | 보안문자 | ✅ | 토지 공시지가 |
| 임대차 신고이력 | 0005 | **공동인증서** | ❌ | 본인 물건만 — 대량수집 부적합 |
| 매매 신고이력·자금조달 | 0005 | **공동인증서** | ❌ | 본인 물건만 — 대량수집 부적합 |

> 신고이력 2종은 공동인증서로 "본인 소유 물건"만 조회 가능하므로 플랫폼 대량 수집 대상이 아니다.
> 향후 "내 계약 관리" 같은 개인화 기능에서 사용자 본인 인증으로 연동할 수 있다.

## 설정 (👤 사용자 액션)

1. developer.codef.io 가입 → 클라이언트 발급.
2. Vercel 환경변수:
   - `CODEF_CLIENT_ID`, `CODEF_CLIENT_SECRET`
   - `CODEF_PUBLIC_KEY` (비밀번호·인증정보 RSA 암호화용)
   - `CODEF_ENV` = `demo` 또는 `api`
3. 설정되면 `isCodefConfigured()`가 true가 되고, 크론/관리자 트리거로 적재가 시작된다.

미설정 시 모든 CODEF 호출은 **정상 폴백**(null/skipped)하며 사이트는 기존 국토부·KB 공개
데이터로 계속 동작한다.

## 코드 구성

- `lib/codef/endpoints.ts` — 12개 상품 카탈로그(명세 코드화).
- `lib/codef/client.ts` — OAuth2 토큰 + 상품 호출, env-gated, graceful.
- `lib/codef/sync.ts` — 응답 정규화 → `public_property_records` 적재(현재 KB 시세 구현).
- `app/api/cron/codef-sync/route.ts` — CRON_SECRET/관리자 보호, `?complex=&serial=` 단건 테스트.
- `lib/market/public-records.ts` — 읽기 전용 로더(단지별·데이터셋별).
- `app/data/records/page.tsx` — 연동 현황 + 단지 자료 조회.
- `app/complex/tx/[slug]` — 단지 상세에 KB 시세 섹션(데이터 있을 때만).
- DB: `public_property_records`(캐시, RLS deny-all=서비스롤 전용) + 조회/중복방지 인덱스.

## 다음 단계(연동 후)

1. `단지 일련번호` 조회로 서울 주요 단지의 KB 단지키 매핑을 캐시.
2. `codef-sync`에 대상 단지 목록을 넣고 크론(주 1회)에 등록.
3. 공시가격·공시지가 정규화 추가(현 sync 패턴 복제).
4. 신고이력은 "내 물건" 개인화 기능으로 별도 설계(본인 인증 UX 포함).
